AMXXPawn编译器指令#file

#file文件名指令使用方法:

#file 字符串字面量

更改脚本文件名称.

从该指令所在行开始,若有导致报错的代码,在展示报错文件名称时,显示该指令设定的字符串字面量.

示例:

#include <amxmodx> #file "C:\MyPlugin.sma" public plugin_init() { #error BAD!!! Baka !!! }

在编译这段源码时,将会看见以下报错:

C:\MyPlugin.sma(5) : fatal error 111: user error: BAD!!! Baka !!!

不论你将脚本保存为什么名称,报错时只会显示"C:\MyPlugin.sma"这个名字.

这个指令在插件运行时依然有效.
