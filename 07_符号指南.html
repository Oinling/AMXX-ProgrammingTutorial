1. 符号类型

AMXXPawn中的符号大致可分为三类：

  • 运算符 - 用于执行各种操作的符号
  • 保留词(关键字) - 语言定义的具有特殊含义的单词
  • 标识符 - 由用户自定义的符号

注意：符号需要先定义，然后才能使用。运算符和保留词是AMXXPawn定义的符号，有固定的作用。标识符是由用户自定义的符号，其用途由用户决定。

2. 运算符

AMXXPawn提供了丰富的运算符，用于执行各种计算和操作：

2.5 逻辑运算符

警告：使用保留词作为标识符名称会导致编译错误。例如：

4. 标识符

标识符是由用户定义的符号，用途由用户决定。

inc文件中包含大量标识符定义，用户引用inc文件的行为，也等于定义了这些标识符。

标识符可以分为以下几种类型：

  • 值类型标签 (Tag)
  • 跳转标签 (Label)
  • 常量 (Constant)
  • 变量 (Variable)
  • 函数 (Function)
  • 参数 (Parameter)
  • 插件状态 (State)
  • 插件自动机 (Automaton)

4.1 标识符命名规则

  • 标识符不能与保留词同名
  • 作用域终点相同的变量不能同名
  • 同一插件状态下的函数名称不能同名（public函数可以与forward函数同名）
  • 在AMXX1.8.3以下版本中，标识符名称不得超过31个字符
  • 在AMXX1.8.2以上版本中，标识符名称不得超过63个字符
  • 标识符区分大小写（例如：foo和Foo是两个不同的标识符）
  • 名称的首字符必须是"_ @ a至z A至Z"这四者之一
  • 名称的后续字符必须是"_ @ a至z A至Z 0至9"这五者之一

4.2 作用域规则

所有标识符都有作用域（能够使用该符号的区域）。如果两个同名变量的作用域终点相同，将导致插件编译失败（编译器无法确定使用哪个变量）。

即使作用域终点不同，在同一个函数内部也应尽量避免定义同名变量，以避免产生歧义。

作用域示例:

// 用static定义名为index的全局变量，作用域终点位于当前文件末尾 static index; // 定义一个函数func public func() { // 用new定义名为index的局部变量，作用域终点位于 } 右花括号 // 与全局变量index的终点不同，不会报错 new index; new var; // 作用域终点位于 } 位置 new var; // 报错！上下两个var的作用域终点相同 return index + var; }

5. 命名规范

良好的命名规范能提高代码可读性和可维护性。以下是AMXXPawn社区中常见的命名约定：

5.1 常量命名

常量通常全部大写，并用下划线分隔单词：

MAX_PLAYERS MAX_NAME_LENGTH

5.2 伪结构体命名

定义伪结构体值类型时，必须以小写字母开头：

enum sPlayer { 结构体属性1索引常量, 结构体属性2索引常量 }

5.3 变量命名

  • 全局变量：通常以"g"开头，每个单词首字母大写
    gZombie gHasWeapon
  • 全局只读变量：每个单词首字母大写
    MaxClients MapName
  • 局部变量：通常以小写字母开头，后续单词首字母大写或全小写用下划线分隔
    playerEntityIndex wpnEntId attackerKills num_args

5.4 函数命名

函数通常全部小写用下划线分隔单词，或每个单词首字母大写：

is_user_alive get_user_weapon ArrayGetCell ExecuteHamB

5.5 常见命名方法

  • 小驼峰命名法：除第一个单词外，其他单词首字母大写
    myQqNumber numPlayers
  • 大驼峰命名法：每个单词首字母大写
    CsInternalModel CsTeams ArrayGetCell GetModelBoundingBox
  • 匈牙利命名法：变量名=值类型缩写+对象描述
    bHasWeapon // b = bool = 布尔型 iCounter // i = int = 整数型 flCounter // fl = float = 浮点型 szMessage // sz = StringWithZeroEnd = 以终止符结尾的字符串
  • 下划线命名法：用下划线分隔单词
    my_name pev_teleport_time set_pev_string get_pdata_int

最佳实践：命名规则应尽可能一致和规范。这能提高代码可读性，使人一看就能猜出符号的大致含义。同时，要避免使用具有歧义的符号名，以防止在开发过程中出现混淆和错误。
